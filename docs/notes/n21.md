# NASH OS Build 18D — Permissioned Action Console Lock

Build 18D replaces informational role homes with action-only permissioned consoles.

## Closed Issue
Previous shells still exposed information panels and drill-down pages without an explicit command. Build 18D changes the landing experience to command-first operation.

## Employee permissions
- Select own record.
- Request profile edit.
- Check in / start workday / check out.
- Load own tasks only on request.
- Start selected task.
- Submit evidence.
- Edit draft evidence.
- Delete draft evidence.
- View own rights / reports.
- View own performance.

## Manager permissions
- Load team queue only on request.
- Add team task request.
- Edit team task request.
- Delete / cancel unapproved task request.
- Review submitted work.
- Accept work.
- Return work.
- Request evidence.
- Escalate SLA.

## Controls
- No data panels load without user request.
- All add/edit/delete operations create runtime receipts.
- Direct database CRUD is blocked.
- MySQL remains source of truth.
- No schema migration.
- No database schema touch.
- Human final decision remains required.
- AI autonomous decisions remain blocked.
