# NASH OS Build 18A — Operational Reality Patch

This patch corrects the UAT finding that Build 18 was technically accepted but still felt informational, role surfaces were too similar, and the product logo was not applied as a mature NASH OS identity.

## What changed

- Added a professional NASH OS Decision Operating System logo in the sidebar.
- Added a role-specific Operational Command Center above the legacy panels.
- Employee, Manager, HR, and Executive now receive different visible operating surfaces.
- Hidden role pages remain protected as drill-down modules and are not deleted.
- Added operational command buttons that dispatch into existing runtime handlers such as attendance, task execution, closure, performance, compensation, government relations, controls, quality, AI radar, and final acceptance.
- Added QA script: `npm.cmd run qa:operational-patch-18a`.

## Controls preserved

- No schema migration.
- No database schema touch.
- MySQL remains source of truth.
- AI cannot approve decisions.
- Human final decision remains required.
