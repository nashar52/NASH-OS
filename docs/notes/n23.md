# IMPLEMENTATION — FINAL GOLD MASTER LOCK

## Decision
Build 18D was not accepted as a product experience because the user still experienced information screens and patch-style behavior. This final lock replaces the visible experience with a strict action-first operating console.

## Final Rules
- No data appears without an explicit command.
- Employee and manager surfaces are separated.
- Employee add/edit/delete are limited to permitted requests and draft evidence.
- Manager add/edit/delete are limited to task requests and manager workflow decisions.
- HR and Executive actions are controlled workflows, not silent database mutations.
- All actions create runtime receipts.
- Direct database CRUD remains blocked in this lock.
- MySQL remains the source of truth.
- No schema migration or database schema touch.
- AI is recommendation/support only; human final decision required.
