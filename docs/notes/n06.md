# NASH OS — Clean Build 07 Employee Self-Service + Rights Lock

## Purpose
Build 07 adds the controlled employee self-service layer on top of the clean operational sequence:

Employee 360 → Workday Attendance → Task Execution → Manager/Beneficiary Closure → JD/SOP Library → Employee Self-Service Rights.

## Added
- Employee Self-Service + Rights panel.
- Load Self-Service View button.
- Export Self-Service Map button.
- Controlled salary summary view.
- Grade / rank, department, position, manager, nationality source labels.
- Personal attendance summary.
- Personal work reports: assigned task candidates, runtime executions, submitted, accepted, returned, closed.
- Self-service view history.
- API endpoints:
  - `/api/self-service/summary`
  - `/api/self-service/rights/:employeeId`
  - `/api/self-service/views/:employeeId`

## Preserved
- Employee picker from MySQL.
- Workday attendance gate.
- Task execution gate.
- Manager / beneficiary closure gate.
- JD + SOP library and AI SOP optimization.

## Governance Constraints
- No schema change.
- No migration.
- No seed.
- No JSON fallback.
- No legacy patch UI.
- No generic operating form.
- Other employee salary records are blocked from normal employee self-service.
- Salary is a controlled summary, not payroll administration.
