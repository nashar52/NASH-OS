# NASH OS — Final Operating System Gold Master

This package closes the UI/UX failure that appeared in Build 18A/18B/18C/18D.

## Final product rule

The application is now an action-first operating console:

- No employee data loads automatically.
- No long stacked information page.
- No shared employee/manager surface.
- Every role sees only its permitted commands.
- Add/Edit/Delete are permissioned runtime requests with receipts.
- Direct database CRUD is blocked in this lock.
- MySQL remains source of truth.
- No schema migration and no database schema touch.
- AI is decision support only; human final decision remains required.

## Roles

Employee: own profile requests, check-in/out, own tasks, evidence drafts, own reports/performance.

Manager: team queue, add/edit/cancel task requests, accept/return submitted work, request evidence, escalate SLA.

HR Operations: employee 360, HR action requests, performance, training, compensation/payroll/WPS, government relations, quality/governance, AI decision support.

Executive: executive brief, risk board, decision backlog, governance risk, AI risk radar, final acceptance.

## Run

```powershell
npm.cmd install
npm.cmd run mysql:lock-check
npm.cmd run qa:clean-build-18
npm.cmd run qa:permissioned-actions-18d
npm.cmd run qa:final-gold
npm.cmd start
```

Open:

```text
http://localhost:3000/?v=final-gold-master
```

## Acceptance

The build is rejected if any screen shows unsolicited employee data, long stacked panels, silent buttons, duplicated role surfaces, or hidden database mutation.
