# NASH OS Build 18C — Single Surface Operational Shell Lock

Purpose: fix the Build 18B issue where role command centers improved the top surface but legacy drill-down panels could still appear stacked in the same scroll page.

What changed:
- Default screen shows only the role-specific Command Center.
- Role home cards, selected-section panel, runtime receipts block, and delivery roadmap are hidden from the runtime operating shell.
- When a command opens a drill-down, the Command Center is temporarily hidden and only that one protected drill-down appears.
- A Back to Command Center control is injected into the active drill-down.
- No database schema change, no migration, no seed, no AI autonomous decision.

Acceptance:
- singleSurfaceShellActive = true
- legacyStackPanelsHidden = true
- oneDrilldownAtATime = true
- backToCommandCenterActive = true
- schemaMigrationIncluded = false
- databaseSchemaTouched = false
