# NASH OS Clean Build 04 — Task Execution Lock

## Purpose
Build 04 adds the first clean operational task execution layer on top of Employee 360 and Workday Attendance.

## Included
- Controlled Employee 360 picker from MySQL remains active.
- Workday attendance remains active: check-in, start session, check-out.
- Task Execution Gate added.
- MySQL `tasks` source inspection added.
- Employee-scoped task source loading added.
- JD/position/department matching signals added where source columns allow it.
- SLA, beneficiary, expected output, and evidence requirements surfaced.
- Start Selected Task action creates runtime execution proof.
- Submit Completion action records action report, evidence reference, output summary, and receipt.
- No generic operating form.
- No legacy patch UI.
- No schema change, migration, seed, or JSON fallback.

## User journey
1. Select a real employee from MySQL.
2. Run PIN Check-in or Face ID Ready.
3. Start Workday Session.
4. Open Task Execution Gate.
5. Load My Tasks.
6. Select a MySQL task.
7. Start Selected Task.
8. Fill action report, evidence reference, and output summary.
9. Submit Completion.
10. Confirm runtime task execution receipt.
